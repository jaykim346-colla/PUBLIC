# 01. 하네스 실습 — 운임 견적서 자동 작성

> 목표: 화주 요청 → 운임 조회 → 금액 계산 → 견적서 완성을
> **항상 똑같은 4단계**로 처리하는 시스템을 만든다.

## 이 방식의 특징

내가 순서를 정해주고, Claude는 **그 순서를 벗어나지 않고** 그대로만 실행합니다.
그래서 누가 몇 번을 돌려도 결과가 동일합니다. 견적서처럼 **실수가 곧 손실**인
업무에 잘 맞습니다.

## 실습 데이터

`harness-practice/data/` 안에 3개:
- `shipper_request.xlsx` — 화주가 보낸 견적 요청 (구간/컨테이너/수량/희망출항일)
- `rate_table.xlsx` — 우리 회사 구간별 단가표
- `quote_template.docx` — `{{REQ_NO}}` `{{TOTAL}}` 같은 빈칸이 있는 견적서 양식

---

## STEP 1. 그대로 복사해서 입력할 프롬프트

Claude Code에 아래를 **그대로** 붙여넣으세요.

```
나는 포워딩 회사 견적 담당자야. 코딩은 몰라.
아래 4단계를 "항상 이 순서 그대로" 실행하는 프로그램을 만들어줘.
중간에 네가 알아서 판단하거나 순서를 바꾸지 말고, 정해진 절차만 따라.

[데이터 위치]
- 화주 요청: harness-practice/data/shipper_request.xlsx
- 운임표:   harness-practice/data/rate_table.xlsx
- 견적서 양식: harness-practice/data/quote_template.docx
  (양식 안의 {{REQ_NO}}, {{SHIPPER}}, {{POL}}, {{POD}}, {{CNTR_TYPE}},
   {{QTY}}, {{ETD}}, {{UNIT_PRICE}}, {{TOTAL}}, {{VALID_UNTIL}}, {{QUOTE_DATE}}
   를 실제 값으로 바꿔 채우면 돼)

[항상 지킬 4단계]
1. shipper_request.xlsx에서 각 요청의 출발항·도착항·컨테이너타입·수량을 읽는다
2. rate_table.xlsx에서 같은 구간+컨테이너타입의 단가와 유효기간을 찾는다
3. 합계 = 단가 × 수량 으로 계산한다
4. quote_template.docx의 빈칸을 채워, 요청 건마다 견적서 Word 파일로 저장한다

요청이 여러 건이면 각각 따로 견적서를 만들어줘.
다 끝나면 어떤 파일이 만들어졌는지 목록으로 알려줘.
```

---

## STEP 2. 결과 확인 (정답 체크리스트)

견적 요청은 3건이므로 견적서도 **3개**가 나와야 합니다. 아래가 맞는지 보세요.

| 요청번호 | 화주 | 구간 | 타입×수량 | 단가 | 합계(정답) |
|---|---|---|---|---|---|
| REQ-2026-001 | 한빛무역 | 부산→LA | 40HC × 2 | 2,150 | **4,300** |
| REQ-2026-002 | 대성물산 | 인천→로테르담 | 20DRY × 3 | 1,450 | **4,350** |
| REQ-2026-003 | 코리아텍 | 부산→싱가포르 | 40DRY × 1 | 950 | **950** |

→ 합계 숫자가 위와 같으면 하네스가 정확히 작동한 것입니다.

---

## STEP 3. "하네스답게" 한번 더 실험하기

하네스의 진가는 **반복해도 똑같다**는 점입니다. 이렇게 해보세요.

1. 위 프롬프트를 한 번 더 실행 → 결과가 동일한지 확인 (같아야 정상)
2. `shipper_request.xlsx`를 열어 수량을 바꾸고 다시 실행
   → 절차는 그대로, 숫자만 바뀐 견적서가 나오면 성공

> 느껴볼 점: 내가 절차를 못 박았기 때문에 Claude가 "마음대로" 하지 않습니다.
> 이것이 하네스의 안정감입니다.

---

## 막힐 때

- 단가를 못 찾는다고 하면 → 요청서의 항만명과 운임표 항만명 표기가 같은지 확인
- 견적서가 안 열린다고 하면 → "방금 만든 견적서를 PDF로도 변환해줘"라고 추가 요청
- 더 욕심내기 → "3건을 요약한 견적 현황표 엑셀도 하나 만들어줘"
