# 02. 에이전트 실습 — 화주 문의 자동 응대 도우미

> 목표: 화주가 보낸 문의를 읽고, **무엇을 묻는지 Claude가 스스로 판단**해
> 알맞은 데이터를 찾아 답하는 도우미를 만든다.

## 이 방식의 특징

이번엔 순서를 정해주지 않습니다. **목표만** 줍니다.
"문의를 보고 알맞게 답해줘." 그러면 Claude가 추적 질문인지, 운임 질문인지,
스케줄 질문인지 **스스로 분류하고**, 필요한 파일을 골라 답을 만듭니다.
매번 들어오는 문의가 다르기 때문에 — 이런 비정형 업무엔 에이전트가 맞습니다.

## 실습 데이터

`agent-practice/data/` 안에:
- `tracking.csv` — 화물 추적 현황 (부킹번호/상태/위치/ETA)
- `schedules.csv` — 선박 스케줄
- `freight_rates.csv` — 선사별 운임
- `inquiries.txt` — 화주 문의 샘플 5종 (추적/운임/스케줄/복합/모호)

---

## STEP 1. 에이전트에게 줄 프롬프트 (그대로 복사)

```
나는 포워딩 회사에서 일해. 코딩은 몰라.
화주들이 보내는 문의는 매번 내용이 달라. 어떨 땐 화물 위치를 묻고,
어떨 땐 운임을, 어떨 땐 스케줄을 물어.

그래서 "문의 내용을 읽고, 무엇을 묻는지 네가 스스로 판단해서"
알맞은 데이터를 찾아 답해주는 도우미 역할을 해줘.
질문 종류를 내가 미리 다 정해줄 수 없으니, 분류와 판단은 네가 알아서 해.

[참고할 데이터]
- 화물추적: agent-practice/data/tracking.csv
- 스케줄:   agent-practice/data/schedules.csv
- 운임:     agent-practice/data/freight_rates.csv

[지켜줘]
- 데이터에 없는 내용은 지어내지 말고 "확인 불가"로 답해
- 문의에 정보가 부족하면(예: 구간을 안 알려줌) 추측하지 말고 되물어
- 답변은 화주에게 바로 보낼 수 있는 정중한 한국어로

준비됐으면 "문의 주세요"라고 말해줘. 내가 화주 문의를 하나씩 줄게.
```

→ Claude가 "문의 주세요"라고 하면, 다음 단계로.

---

## STEP 2. 문의를 하나씩 던지기

`inquiries.txt`에 있는 문의를 **한 번에 하나씩** 복사해 입력하세요.
각 문의에서 에이전트가 어떻게 '판단'하는지 관찰하는 게 핵심입니다.

| # | 문의 | 에이전트가 해야 할 판단 | 정답 방향 |
|---|---|---|---|
| 1 | "HMMU1234567 지금 어디?" | 추적 질문이라 인식 → tracking.csv | 태평양 해상, ETA 7/18 |
| 2 | "부산→LA 40HC 운임, 제일 싼 곳?" | 운임 질문 → freight_rates.csv | ONE 1,980이 최저 |
| 3 | "부산-싱가포르 제일 빠른 배?" | 스케줄 질문 → schedules.csv | PIL, 6일 직항 |
| 4 | "코리아텍 화물 상태 + 마감 임박?" | 복합 → 추적 조회 후 경고 | MSCU…, 서류마감 임박 안내 |
| 5 | "운임 좀 알려주세요." | **정보 부족 → 되물어야 정상** | "어느 구간/컨테이너?"라고 질문 |

> 가장 중요한 건 **문의 5번**입니다.
> 에이전트가 멋대로 아무 운임이나 답하면 실패, **되물으면 성공**입니다.
> 이게 "알아서 판단한다"의 핵심 — 모를 땐 묻는 판단까지 포함합니다.

---

## STEP 3. "에이전트답게" 실험하기

1. **같은 문의를 두 번** 넣어보기 → 표현이 조금씩 달라질 수 있음
   (하네스와 정반대 — 유연함의 증거)
2. inquiries.txt에 **없는 새 문의**를 직접 지어 던지기
   예: "한빛무역 화물 두 개 다 어떻게 되고 있어요?"
   → 에이전트가 tracking.csv에서 한빛무역 2건을 알아서 찾아 답하면 성공
3. 일부러 함정 주기: "부킹번호 ZZZZ9999999 어디 있어요?"
   → 데이터에 없으므로 "확인 불가"로 답해야 정상 (지어내면 실패)

---

## 두 실습 비교 포인트

| 관찰 항목 | 하네스(견적서) | 에이전트(문의응대) |
|---|---|---|
| 같은 입력 반복 시 | 항상 동일 | 조금씩 다를 수 있음 |
| 예상 못 한 입력 | 처리 못 함 | 알아서 분류 시도 |
| 통제권 | 내가 가짐 | Claude에 위임 |
| 안심 vs 편함 | 안심됨 | 편함 |

→ 직접 느낀 이 차이가, 내 업무에 **어떤 방식을 언제 쓸지** 고르는 기준이 됩니다.
