# 물류업무 시스템 만들기 체험 가이드
### 비개발자를 위한 하네스 vs 에이전트 실습

코딩을 몰라도 괜찮습니다. **Claude Code(또는 Claude 채팅창)에 한국어로 요청만 하면**
내 업무를 돕는 작은 시스템이 만들어집니다. 이 체험에서는 두 가지 방식을 직접 비교합니다.

---

## 들어가기 전에: 두 방식의 차이

| | **하네스 방식** | **에이전트 방식** |
|---|---|---|
| 한 줄 정의 | 내가 **순서를 정해주고** 그대로 실행 | 목표만 주면 **알아서 판단**해 실행 |
| 결과 | 매번 **똑같음** (예측 가능) | 상황 따라 **유연** |
| 비유 | 자판기 (버튼=정해진 결과) | 비서 (알아서 처리) |
| 적합한 업무 | 반복되는 정형 업무 | 매번 다른 비정형 업무 |
| 이번 실습 | **예시1: 운임 견적서 자동 작성** | **예시2: 화주 문의 자동 응대** |

> 핵심 체감 포인트: 같은 도구(Claude)인데, **내가 통제하느냐 / 맡기느냐**에 따라
> 결과물의 성격이 완전히 달라진다는 걸 직접 느끼는 것이 이 체험의 목적입니다.

---

## 준비물

`logistics-lab` 폴더 안에 실습 데이터가 모두 들어 있습니다.

```
logistics-lab/
├── harness-practice/data/     ← 하네스 실습용
│   ├── shipper_request.xlsx   (화주 견적요청 3건)
│   ├── rate_table.xlsx        (구간별 운임표)
│   └── quote_template.docx    (견적서 빈 양식)
│
└── agent-practice/data/       ← 에이전트 실습용
    ├── tracking.csv           (화물추적 현황)
    ├── schedules.csv          (선박 스케줄)
    ├── freight_rates.csv      (선사별 운임)
    └── inquiries.txt          (화주 문의 샘플 5종)
```

자세한 진행 방법은 두 개의 세부 가이드를 순서대로 보세요:
- `01_하네스_실습.md`
- `02_에이전트_실습.md`

---

## 추천 진행 순서 (약 40분)

1. (5분) 위 표로 두 방식 차이 이해
2. (15분) **하네스 실습** — 견적서 자동 작성 (`01_하네스_실습.md`)
3. (15분) **에이전트 실습** — 문의 자동 응대 (`02_에이전트_실습.md`)
4. (5분) 아래 '마무리 토론'으로 차이 정리

---

## 마무리 토론 (체험 후 함께 이야기)

- 내 실제 업무 중 **"매번 똑같은 순서로 하는 일"** 은? → 하네스로 만들 후보
- 내 업무 중 **"매번 상황 봐서 판단하는 일"** 은? → 에이전트로 만들 후보
- 하네스가 더 안심됐던 점 / 에이전트가 더 편했던 점은?
- 우리 팀에 가장 먼저 도입하고 싶은 자동화 1가지는?
